# AML 의심 신호 정의 (AML Suspicious Activity Signals)

## 목적

이 문서는 본 프로토타입이 탐지하는 자금세탁(AML) 의심 신호의 **정의, 의심 근거,
탐지 기준, 필요한 데이터 필드**를 정리한다. LLM 기반 검토 리포트 생성 시, 모델은
이 문서에 정의된 신호 기준을 근거로 의심 거래 후보와 그 사유를 제시한다.

## 전제 및 한계

- 본 문서의 임계값은 **시연용 예시값**이며 실제 규제 기준과 다르다.
  (예: 특금법 트래블룰 적용 기준은 100만원 상당 이상의 가상자산 이전)
- 본 프로토타입은 **1차 스크리닝 보조 도구**이며, 최종 의심거래 판단과
  STR(의심거래보고) 여부는 준법감시 담당자가 결정한다.
- 모든 입력은 **더미 데이터**이며 실제 고객·거래 정보를 사용하지 않는다.
- 오탐(false positive)을 전제로 설계한다. 신호 탐지 = 의심거래 확정이 아니라,
  "담당자가 추가 확인할 후보"를 의미한다.

## 데이터 모델

### accounts (계정 단위 KYC 프로필)

| 필드 | 설명 |
|------|------|
| `account_id` | 계정 식별자 |
| `account_age_days` | 계정 개설 후 경과 일수 |
| `kyc_income_band` | 신고 소득 구간 (`under_30m`, `30m_50m`, `50m_100m`, `100m_plus`) |
| `occupation` | 신고 직업 |

### transactions (거래 단위)

| 필드 | 설명 |
|------|------|
| `transaction_id` | 거래 식별자 |
| `account_id` | 거래 계정 |
| `timestamp` | 거래 시각 (ISO 8601) |
| `type` | `deposit` / `withdrawal` / `trade` |
| `amount_krw` | 거래 금액의 원화 환산액 |
| `asset` | 자산 종류 (`KRW`, `BTC`, `ETH` 등) |
| `counterparty_address` | 입출금 상대 지갑 주소 (법정화폐·매매는 null) |
| `counterparty_risk` | 상대 주소 위험도 (`none`, `mixer`, `darknet`, `sanctioned`, `flagged`) |

## 신호 정의

### S1. 구조화 거래 (Structuring)

- **정의**: 단시간 내 보고 임계금액 바로 아래 금액으로 입출금을 반복.
- **의심 근거**: 고액거래보고(CTR)·의심거래 기준 회피를 위한 전형적 분할 행위.
- **탐지 기준 (예시)**: 동일 계정이 24시간 내, 보고 임계금액(예시 1,000만원)의
  80~99% 구간 거래를 **3건 이상** 발생.
- **필요 필드**: `amount_krw`, `timestamp`, `type`, `account_id`

### S2. 통과 계좌 패턴 (Pass-through)

- **정의**: 입금 직후 거의 전액을 단시간 내 출금하고, 그 사이 매매 활동이 없음.
- **의심 근거**: 거래소를 자금 세탁의 경유지(layering)로 사용하는 행위.
- **탐지 기준 (예시)**: 입금 후 6시간 이내에 입금액의 **90% 이상**을 출금,
  해당 기간 내 `trade` 거래 0건.
- **필요 필드**: `type`, `timestamp`, `amount_krw`, `account_id`

### S3. 신규 계정 비정상 대량 거래

- **정의**: 개설 직후의 계정이 프로필 대비 과도한 거래량을 발생.
- **의심 근거**: 명의 도용·차명 계정·일회성 세탁 계정의 가능성.
- **탐지 기준 (예시)**: `account_age_days <= 7` 이면서 단일 또는 누적 거래액이
  기준(예시 5,000만원)을 초과.
- **필요 필드**: `account_age_days`, `amount_krw`, `account_id`

### S4. 고위험 주소 연관

- **정의**: 믹서/텀블러, 다크넷, 제재 대상, 기등록 위험 지갑과의 입출금.
- **의심 근거**: 자금 출처·목적지의 익명화 또는 제재 위반 가능성.
- **탐지 기준**: `counterparty_risk` 값이 `mixer`, `darknet`, `sanctioned`,
  `flagged` 중 하나.
- **필요 필드**: `counterparty_address`, `counterparty_risk`

### S5. KYC 프로필 불일치

- **정의**: 신고 소득·직업 등 CDD 정보 대비 거래 규모가 과도.
- **의심 근거**: 자금 출처 불명, 제3자 자금의 대리 거래 가능성.
- **탐지 기준 (예시)**: 누적 거래액이 신고 소득 구간 상한의 **3배**를 초과.
- **필요 필드**: `kyc_income_band`, `occupation`, `amount_krw`

## 위험도 산정 (예시 규칙)

| 위험도 | 조건 |
|--------|------|
| **High** | 고위험 주소 연관(S4) 발생, 또는 신호 2개 이상 동시 충족 |
| **Medium** | 단일 신호(S1·S2·S3·S5 중 하나) 충족 |
| **None** | 충족 신호 없음 → "특이사항 없음" 처리 |

## 정상(미탐지) 처리 원칙

신호에 해당하지 않는 계정은 **반드시 "특이사항 없음"으로 명시 출력**한다.
모든 거래를 의심으로 분류하지 않는 것은 본 도구의 신뢰성과 오탐 관리의 핵심이다.
리포트에는 "왜 정상으로 판단했는지"의 근거도 간단히 포함한다.

## 다음 단계 (프로토타입 범위 밖)

- 실제 규제 기준값 반영 및 임계값 튜닝
- 시계열·네트워크 기반 패턴 탐지 추가 (분산 이동, 순환 거래 등)
- 외부 위험 주소 DB(체이널리시스 등) 연계
- 운영 데이터 적용 시 개인정보 국외이전·망분리 검토
