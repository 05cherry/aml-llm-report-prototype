# CLAUDE.md

## 작업 원칙 (모든 작업에 우선 적용)

LLM 코딩의 흔한 실패를 줄이기 위한 4원칙. 세부 지시와 충돌하면 이 원칙이 우선한다.

1. **코딩 전 사고 (Think Before Coding)**
   가정을 멋대로 세우고 달리지 말 것. 모호하거나 정보가 부족하면 먼저 가정을
   명시하고, 상충·트레이드오프를 드러내고, 필요하면 질문해 확인한 뒤 진행한다.

2. **단순함 우선 (Simplicity First)**
   문제를 푸는 최소한의 코드만 작성한다. 추측성 기능, 불필요한 추상화, 과도한
   일반화를 만들지 않는다. 100줄로 될 일을 1000줄로 만들지 않는다.

3. **외과적 변경 (Surgical Changes)**
   작업에 필요한 부분만 수정한다. 과제와 무관한 코드·주석은 건드리지 않으며,
   충분히 이해하지 못한 코드는 임의로 바꾸거나 지우지 않는다.

4. **목표 기반 실행 (Goal-Driven Execution)**
   아래 "완료 기준"을 충족할 때까지 스스로 반복·검증한다. 기준을 만족하면
   멈추고, 시키지 않은 범위로 넓히지 않는다.

## 프로젝트

준법감시(AML) 1차 검토를 보조하는 LLM 기반 리포트 생성 프로토타입.
입력: 더미 거래 데이터(JSON) → 출력: 계정별 AML 검토 리포트(Markdown).
**면접용 데모이며, 의도적으로 핵심 기능만 구현한다.**

## 기술 스택

- Python 3.13 (venv)
- Anthropic Claude API + `anthropic` SDK, 모델은 Sonnet
- 의존성 최소화: `anthropic`, `python-dotenv` 수준.
  새 의존성을 추가하기 전에 먼저 이유를 설명하고 확인을 받을 것.

## 디렉토리

- `src/` : CLI 진입점, Claude API 호출, 리포트 생성, 신호 판정 로직
- `data/sample_transactions.json` : 입력 더미 데이터
- `docs/aml-signals.md` : AML 신호 정의 (도메인 규칙의 단일 진실 공급원)
- `docs/prompt-design.md`, `docs/limitations.md` : 설계 노트
- `sample_report.md` : 대표 실행 결과물
- `screenshots/` : 실행 화면

## 도메인 규칙 (중요)

- AML 신호의 정의·임계값·위험도 규칙은 **반드시 `docs/aml-signals.md`를 기준**으로 한다.
  코드에 임계값을 임의로 만들어 넣지 말 것. 문서와 코드가 다르면 문서가 우선이며,
  변경이 필요하면 문서를 먼저 고친 뒤 코드를 맞춘다.
- 신호에 해당하지 않는 계정은 **반드시 "특이사항 없음"으로 명시 출력**한다.
- 모든 의심 판정에는 근거(어떤 신호 / 어떤 거래·필드)를 함께 출력한다.

## 출력 구조

계정별 리포트는 다음 4개 항목을 포함한다:
의심 신호 후보 / 탐지 근거 / 추가 확인 항목 / 위험도.
형식 일관성은 프롬프트 지시가 아니라 **tool(구조화 출력) 스키마로 강제**한다.

## 하지 말 것

- API 키 하드코딩 금지. 반드시 `.env`에서 로드. `.env`는 커밋 금지.
- 웹 UI, DB, 인증, 비동기 큐 등 범위 밖 기능 추가 금지. CLI로 충분하다.
- 실제 규제 기준값·외부 위험주소 DB 연동을 임의로 추가하지 말 것 (프로토타입 범위 밖).
- 더미 데이터 외 실데이터 처리 코드를 작성하지 말 것.

## 작업 흐름 (GitHub 이슈 연동)

- 초기 1회: `/sync-plan`으로 마일스톤·이슈·보드를 생성한다.
- 이후 각 작업은 가능하면 `docs/project-plan.md`의 이슈에 대응시킨다.
  - 작업 착수 시: 해당 이슈를 Projects 보드의 `In progress`로 옮긴다.
  - 작업 완료 시: 한 일을 이슈에 코멘트로 남기고, 완료되면 이슈를 닫고 `Done`으로 옮긴다. (gh 사용)
- **매 작업을 마칠 때마다** 결과물과 함께 "어느 이슈에 반영했는지"를 보고한다.
  대응하는 이슈가 없는 새 작업이면, 진행 전에 "이슈로 만들까요?"라고 사용자에게 먼저 묻는다.
- 이슈·보드는 생성·코멘트·상태변경만 한다. 임의 삭제·수정 금지.

## 완료 기준 (성공 조건)

1. 단일 CLI 명령으로 `data/sample_transactions.json`을 입력받아 5개 계정 리포트를 생성한다.
2. ACC-A~ACC-D는 각자 해당하는 신호와 근거가 리포트에 나타난다.
3. ACC-E는 "특이사항 없음"으로 출력된다.
4. 출력 형식이 5개 계정 모두에서 동일한 구조를 유지한다.
